Ayuda y acompañamiento

Soporte

Un espacio para resolver problemas sin vueltas: instalación, accesos, roles, errores visuales, carga de materiales y acompañamiento durante la etapa beta.

Principio guía

Un buen soporte empieza con un buen reporte.

Mientras más claro sea el problema, más rápido se puede resolver. Captura, modelo de celular, versión de Android y pasos para repetir el error ayudan muchísimo.

Instalación

Orientación para descarga, permisos y apertura de la APK en Android.

Accesos

Revisión de usuario, comunidad, provincia y rol asignado.

Errores

Registro claro de fallas para corregirlas sin perder tiempo.

Consultas técnicas

Para problemas de instalación, acceso, descarga de APK, errores visuales o fallas de funcionamiento, se recomienda registrar el modelo del celular, versión de Android, captura de pantalla y descripción concreta del problema.

Problemas de acceso

Si un usuario no puede ingresar o no visualiza los módulos correspondientes, debe verificarse primero su correo, rol asignado, provincia, comunidad y estado de aprobación dentro del sistema.

Reporte de errores

Un buen reporte debería incluir qué acción se estaba realizando, qué se esperaba que ocurriera, qué ocurrió realmente y si el error se repite.

Este criterio permite diferenciar entre una falla real, un problema de conexión, una confusión de uso o un permiso mal asignado.

Solicitudes de contenido

Los pedidos de carga de materiales, noticias, avisos o documentos deben canalizarse según la estructura de roles definida para evitar duplicaciones o publicaciones incorrectas.

Contacto sugerido

Durante la etapa beta, el soporte puede centralizarse en el equipo responsable de implementación de la app. Más adelante puede añadirse aquí un correo, formulario o canal oficial de asistencia.

Nos edificamos juntos

El Soporte requiere darnos para que este proyecto siga avanzando.

La app será mejor si quienes la usan pueden comunicar problemas de forma clara y si el equipo de implementación responde con criterio y continuidad.
